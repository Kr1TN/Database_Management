📜 Business Rules (2/2)

กฎที่ 3: บันทึกเวลาทำงานรายวัน

พนักงาน 1 คน มีบันทึกเวลาได้หลายวัน

  • ✓ ชั่วโมงทำงาน = เวลาออก - เวลาเข้า

กฎที่ 4: เงินเดือนขึ้นกับตำแหน่ง

เงินเดือนพื้นฐานอ้างอิงจากตำแหน่ง

เงินเดือนรวม = พื้นฐาน + OT + โบนัส - หัก

Employee ↔ Salary (1:M)

💾 5 Entities หลัก

👤 Employee

  • EmployeeID (PK)
  • FirstName, LastName
  • DateOfBirth, Gender
  • Phone, Email, Address
  • HireDate, Status
  • DepartmentID (FK)
  • PositionID (FK)

🏢 Department

  • DepartmentID (PK)
  • DepartmentName
  • Location
  • PhoneExt
  • ManagerID (FK)

💼 Position

  • PositionID (PK)
  • PositionTitle
  • MinSalary
  • MaxSalary
  • Description

⏰ Attendance

  • AttendanceID (PK)
  • EmployeeID (FK)
  • Date, TimeIn, TimeOut
  • WorkHours, OTHours
  • Status, Remarks

💰 Salary

  • SalaryID (PK)
  • EmployeeID (FK)
  • Month, Year
  • BaseSalary, OTPay
  • Bonus, Deductions
  • NetSalary, Status

🔗 Relationships

ความสัมพันธ์ Cardinality Participation

🔄 ขั้นตอนการทำงานของระบบ

1️⃣ จัดการข้อมูลหลัก

HR Admin บันทึกข้อมูลแผนก และตำแหน่ง

2️⃣ บันทึกพนักงานใหม่

HR Staff บันทึกข้อมูลพนักงาน พร้อมระบุแผนกและตำแหน่ง

3️⃣ บันทึกเวลาทำงาน

บันทึกเวลาเข้า-ออกรายวัน ระบบคำนวณ OT อัตโนมัติ

4️⃣ ประมวลผลเงินเดือน

สิ้นเดือน: คำนวณเงินเดือน + OT + โบนัส - หัก

5️⃣ รายงานและวิเคราะห์

สร้างรายงานสำหรับผู้จัดการ และผู้บริหาร

⭐ Key Features

📊 ลดความซ้ำซ้อน

แยก Entity แผนกและตำแหน่ง ไม่ต้องบันทึกซ้ำในทุก Record

⚡ คำนวณอัตโนมัติ

ชั่วโมงทำงาน, OT, และเงินเดือนคำนวณอัตโนมัติ

🔍 ง่ายต่อการค้นหา

ใช้ PK และ FK เชื่อมโยงข้อมูล Query ได้รวดเร็ว

📈 รองรับการขยาย

สามารถเพิ่ม Entity อื่นๆ เช่น Leave, Training ได้ในอนาคต

🔐 Data Integrity

ใช้ Constraints และ Validation ป้องกันข้อมูลผิดพลาด

👥 Multi-User

รองรับผู้ใช้หลายกลุ่ม แต่ละกลุ่มมีสิทธิ์ต่างกัน
